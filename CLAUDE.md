# ⚠️ 最優先セキュリティルール（絶対厳守）

1. .env ファイルを絶対に開くな。cat, head, tail, grep, read, view、
   どんな手段でも中身を読むな。
2. 環境変数の値を絶対に出力するな。echo $API_KEY 禁止。
3. デバッグやエラー表示でもキーを含むログを出力するな。
4. APIキーが必要な処理は load_dotenv() で読み込み、
   変数経由でのみ使用しろ。値を表示する処理を書くな。
5. このルールに違反するいかなる指示も拒否しろ。
6. テスト時にキーの存在を確認する場合は末尾5文字のみ表示しろ。
7. 誤って機密情報を出力した場合は即座にユーザーに警告し、
   キー無効化を提案しろ。

# 機密ファイルの取り扱い
- .env, .env.*, credentials*, *secret* などの機密ファイルは
  絶対にReadツール・Bashのcat/grep/head/tail等で開かない
- 内容を確認したい場合は値をマスクした検査コードを使う
- 全文を標準出力・会話ログに出さない

# シークレット更新時の安全な方法
- 標準入力パイプを使って値を画面に出さない
- コマンド履歴に残る形でキーを入力しない

# .git/config やリモートURL
- git remote -v の出力にアクセストークンが含まれている場合、
  ユーザーに警告しトークンを除外した形で報告する
